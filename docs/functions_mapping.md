# Маппинг функций и модулей

| Пункт ТЗ | Модуль | Описание реализации |
| --- | --- | --- |
| 4.1.1 Управление объектами | `app/api/routers_objects.py`, `app/models.py`, `app/schemas.py` | CRUD для ПС, РУ, ячеек с асинхронным доступом и аудитом |
| 4.1.2 Шкафы и устройства | `app/api/routers_panels.py`, `app/models.py`, `app/schemas.py` | Учёт шкафов РЗА, их устройств, статусов и привязки к ячейкам |
| 4.1.2 Конфигурации и уставки | `app/api/routers_configs.py`, `app/audit.py` | Версионность DeviceConfig, SettingRevision, активация ревизий |
| 4.2.1 Управление объектами РЗА | `app/api/routers_objects.py`, `app/db.py` | Асинхронный доступ к данным, интеграция с БД |
| 4.2.2 Конфигуратор устройств | `app/api/routers_panels.py`, `app/api/routers_configs.py` | API для настройки шкафов, устройств и конфигураций |
| 4.2.3 Документооборот | `app/models.py`, `app/schemas.py` (Document) | Модель документов с контролем целостности |
| 4.2.4 Отчётность и аналитика | `app/api/routers_reports.py` | Сводные отчёты, дерево ПС, история конфигураций |
| Аудит критичных операций | `app/audit.py`, использование во всех роутерах | Логирование действий с before/after состоянием |
| Конфигурация и параметры | `app/config.py` | Управление параметрами сервиса, CORS, БД |
